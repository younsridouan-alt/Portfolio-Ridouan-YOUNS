🚀 Pipeline ETL de Données de Ventes – AdventureWorks2019

📅 Période Septembre 2024 – Décembre 2024 Projet universitaire (Master 2)

🎯 Problématique

Mettre en place un pipeline de traitement de données robuste pour centraliser, transformer et intégrer des données hétérogènes (relationnelles, fichiers plats, API) dans un entrepôt de données unifié. L’objectif est de permettre une analyse consolidée des ventes dans un environnement scalable et automatisé.

Stack Technique

Base de données source : SQL Server (AdventureWorks2019) Sources complémentaires : Fichiers CSV, Parquet, API météo REST ETL / Orchestration : Python, Jupyter Notebook Stockage / DWH : Datawarehouse dimensionnel Conteneurisation : Docker Environnement : Windows, VS Code Langages / Librairies : Pandas, SQLAlchemy, Requests, pyodbc

Résultats Mesurables

+30%
+la réactivité des analyses/décisions
< 1 seul point d'accès
aux données via un entrepôt propre et structuré
Processus global :

ingetsion de données -> Transformation des données -> Modélisation & Stockage -> Automatisation & Processing 


Extraction multi-sources :

 - Données transactionnelles extraites de SQL Server via pyodbc

 - Fichiers CSV et Parquet traités avec Pandas

 - Données météo extraites d’une API REST (format JSON) avec la librairie requests

Transformation des données :

 - Nettoyage (NA, formats de dates, encodages)

 - Normalisation et enrichissement via jointures multi-tables

 - Agrégation des ventes par période, produit et région

 - Intégration de données météo pour analyse de corrélation externe

Modélisation & Stockage :

 - Schéma en étoile dans un datawarehouse dimensionnel

 - Création de dimensions produit, client, temps, météo, et d’une table de faits "ventes"

Automatisation & Conteneurisation :

 - Exécution automatique des notebooks avec papermill

 - Pipeline packagé et déployé avec Docker pour portabilité et exécution sur différents environnements
📑 Documentation du Pipeline

Rédaction d'une documentation technique claire du pipeline : architecture, choix technologiques, schémas de flux, exemples de données. Livrable à destination d'un profil technique (développeur, analyste ou DataOps).

🧍‍♂️ Analyse Descriptive & Prédictive de la Posture d’un Opérateur Industriel

📅 Période Avril 2024 – Juin 2024 Projet universitaire (Stage – Master 2) – Équipe de 3 personnes

Problématique

Les troubles musculosquelettiques (TMS) représentent une cause majeure d’accidents du travail dans le secteur industriel. L’objectif de ce projet est de concevoir une solution de surveillance intelligente de la posture à partir de données collectées sur le terrain, en appliquant des techniques de data science et de machine learning pour identifier les postures à risque et anticiper les situations dangereuses.

Stack Technique

Fichiers CSV issus de capteurs de posture Pandas, NumPy Scikit-Learn (Random Forest, SVM, KNN), GridSearchCV Norme REBA (Rapid Entire Body Assessment) Jupyter Notebook, VS Code Microsoft Excel, Markdown

Résultats Mesurables

87% avec Random Forest
🧠 Précision du modèle
< 0.5 points
📉 Écart type des erreurs REBA+
Les grandes étape de projet

🔍 Analyse Descriptive :
Nettoyage, traitement des valeurs manquantes et normalisation des données angulaires (Python/Pandas).

Étude exploratoire des postures : histogrammes, heatmaps, distribution des angles de segments corporels.

Identification de patterns récurrents selon les horaires, la fréquence ou la durée de certaines postures.

📏 Référentiel & Étiquetage :
Application de la norme REBA pour classifier les postures selon leur risque ergonomique.

Création d’un système d’étiquettes de niveau de risque (faible, modéré, élevé) pour chaque observation.

🤖 Analyse Prédictive :
Entraînement de modèles de classification (SVM, KNN, Random Forest) pour prédire la catégorie de risque REBA à partir des capteurs.

Évaluation via cross-validation, matrice de confusion, accuracy, recall, f1-score.

Meilleur score obtenu : Accuracy > 87% avec Random Forest, hyperparamétré avec GridSearchCV.

📈 Visualisation :
Génération de courbes de performance (ROC, learning curves) pour expliquer les résultats au client final.

Mise en forme des recommandations sous forme de dashboards statiques pour la direction QHSE.
Innovation : Annotation Semi-Automatique

Mise en place d’une méthode de pré-annotation automatisée des données via REBA calculée dynamiquement, afin d’accélérer l’étiquetage manuel en réduisant la charge de travail de l’équipe.

🧠 Amélioration de la Qualité des Données dans les Systèmes de Process Mining par l’Intelligence Artificielle

📅 Période Février 2025 – Juin 2025 Projet de mémoire de fin d’études – Master 2 S2IN Encadrant : Mr. Hervé Verjus

Problématique

Dans les systèmes de Process Mining, l’intégration de modèles d’intelligence artificielle est de plus en plus courante pour prédire les comportements futurs, automatiser des décisions ou détecter des anomalies Cependant, la qualité des données reste un facteur critique : les logs sont souvent bruités, incomplets ou déséquilibrés. Ce projet cherche à répondre à la question : Comment améliorer la qualité des données dans les systèmes de Process Mining intégrant l’IA afin d’optimiser les performances prédictives et garantir l’équité algorithmique ?

📎 Télécharger le poster de présentation (Google Drive)

Stack Technique

BPI Challenge 2016 (Pays-Bas) – 65k événements, 26k cas (clics, réclamations, messages...) Nettoyage, enrichissement linguistique, TF-IDF, normalisation Oversampling (SMOTE-like) pour classes rares (ex : réclamations) Heuristics Miner, PCA, clustering non-supervisé Random Forest Classifier, évaluation comparative phase 1 / phase 2 Python, Scikit-Learn, Pandas, Matplotlib, ProM, Jupyter Notebook

Résultats Mesurables

0.46 → 0.78 (gain de +32 pts)
🎯 F1-Score sur les réclamations
71% → 85%
🧠 Précision globale du modèle
89% → 65%
📉 Taux de biais sur la classe dominante (meilleure équité interclasse)


🧪 Terrain expérimental :
Données multilingues et bruitées issues d’un système client complexe

Reconstruction du processus client :
Soumission → Vérification → Message Werkmap ↔ Réponse client → Réclamation / Clôture

📉 Phase 1 – Données brutes :
Extraction directe des logs

PCA montre des clusters flous et déséquilibrés

Modèle Random Forest biaisé vers la classe majoritaire (précision faible sur les réclamations)

🧼 Phase 2 – Pipeline amélioré :
Nettoyage sémantique des descriptions multilingues

Enrichissement avec TF-IDF + features d’interaction client

Rééquilibrage via oversampling sur les intentions rares

Résultat : augmentation de la F1-Score moyenne et meilleure séparation des clusters

🔬 Contributions Scientifiques
✅ Une démonstration empirique de l’impact de la qualité des logs sur la performance des modèles IA.

✅ Un pipeline reproductible combinant nettoyage, enrichissement vectoriel, oversampling.

✅ Une base de travail pour des systèmes de Process Mining plus justes et prédictifs.

💼 Recommandations Managériales
Intégrer un pipeline de qualité en amont dans tout projet de Process Mining.

Prioriser les intentions rares (ex. réclamations) dans les modèles de classification.

Mettre en place des contrôles d’équité algorithmique pour des décisions explicables et auditées.

☁️ Analyse de l’Impact du Tourisme sur l’Économie Territoriale d’Annecy via un Pipeline ETL Cloud-Natif

📅 Période Juillet 2025 – En cours Projet personnel de spécialisation – Data Engineering & Analyse Territoriale (GCP)

Problématique

Le tourisme est un moteur essentiel de l’économie locale à Annecy, mais son impact réel sur les variables socio-économiques territoriales (emploi, logement, mobilité, activité commerciale) reste difficile à quantifier. Ce projet vise à construire un pipeline de données cloud-native sur GCP pour agréger et analyser différentes sources (tourisme, météo, économie, logement) afin de produire des indicateurs fiables, dynamiques et visualisables pour les acteurs publics et privés..

Stack Technique

Google Cloud Platform (GCP) Cloud Storage, Cloud Pub/Sub, API publiques (Open Data, Insee, Météo) Dataflow (Apache Spark), Cloud Functions, Python BigQuery (entrepôt de données analytiques) Cloud Composer (Airflow) Stackdriver (Cloud Monitoring & Logging)
